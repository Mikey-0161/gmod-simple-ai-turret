# Contributing Guidelines

First off, thank you for taking the time to contribute to this project! Every contribution, whether it's a bug report, feature suggestion, documentation improvement, or code submission, is appreciated.

## Before You Start

Before creating an issue or pull request:

* Make sure you're using the latest version of the addon.
* Search existing Issues to see if your problem or suggestion has already been reported.
* Keep discussions respectful and follow the project's Code of Conduct.

## Reporting Bugs

When reporting a bug, please include as much information as possible:

* A clear description of the issue.
* Steps to reproduce it.
* What you expected to happen.
* What actually happened.
* Any error messages from the console.
* Your Garry's Mod version and any relevant addons installed.

Providing screenshots or videos can also be helpful.

## Suggesting Features

Feature requests are always welcome.

When suggesting a feature, please explain:

* The problem you're trying to solve.
* Your proposed solution.
* Any alternative solutions you've considered.

Please understand that not every suggestion will be implemented, but all reasonable ideas will be considered.

## Submitting Pull Requests

Before submitting a Pull Request:

* Keep changes focused on a single feature or fix.
* Test your changes thoroughly before submitting.
* Follow the existing code style where possible.
* Remove unnecessary debug code or comments.
* Update documentation if your changes affect how the addon works.

## Commit Messages

Clear commit messages make reviewing changes much easier.

Examples:

* Fix turret targeting through walls
* Add configurable firing delay
* Improve NPC detection performance
* Update README installation instructions

Avoid vague commit messages such as:

* Update
* Fix stuff
* Changes

## Coding Style

Please try to:

* Write clean and readable code.
* Use descriptive variable and function names.
* Keep functions reasonably small.
* Add comments only where they improve understanding.
* Avoid unnecessary complexity.

## Questions

If you're unsure about something before starting work, feel free to open an Issue to discuss it first.

## Thank You

Thank you for helping improve this project. Every contribution helps make it better for everyone.
