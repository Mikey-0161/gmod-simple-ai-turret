# Security Policy

## Supported Versions

Security updates are only provided for the latest released version of this project.

| Version            | Supported |
| ------------------ | --------- |
| Latest Release     | ✅         |
| Older Releases     | ❌         |
| Development Builds | ❌         |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** create a public GitHub Issue.

Instead, report it privately by using GitHub's **Private Vulnerability Reporting** feature if it is enabled, or contact the project maintainer directly.

When submitting a report, please include:

* A clear description of the vulnerability.
* Steps to reproduce the issue.
* The affected version.
* Any proof-of-concept or screenshots, if applicable.
* Your suggested fix (optional).

## Response Timeline

While response times may vary, the goal is to:

* Acknowledge reports within **7 days**.
* Investigate and validate the vulnerability.
* Develop and test a fix if the report is confirmed.
* Release a security update as soon as reasonably possible.

## Disclosure Policy

Please allow time for the vulnerability to be investigated and fixed before publicly disclosing it. Responsible disclosure helps protect everyone using this project.

## Scope

This policy covers security issues within this project's source code.

General bugs, feature requests, and gameplay issues should be reported through the GitHub Issues page instead.

## Thank You

Thank you for helping keep this project secure. Responsible security reporting is greatly appreciated and helps improve the project for everyone.
